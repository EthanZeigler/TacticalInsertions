Welcome to Tactical Insertions!

Tactical Insertions is the result of a plugin request on Bukkit.org by GreenPls. Remember the tactical insertion from Call of Duty Black Ops as well as Modern Warfare? Tactical Insertions does exactly this! This plugin allows your server's players to place blocks that act just as Call of Duty's tactical insertion. When a player dies, they respawn at the location of the insertion. However, what if this "breakable warp" concept could be used to create multiple warps? This is what makes Tactical Insertions so impressive.

Why Two Modes?

When this plugin idea was originally introduced by GreenPls, I saw the opportunity for 2 modes to be implemented-- a warp mode and a respawn mode. The respawn mode is what most people think of when they imagine the Call of Duty insertion. When the player dies, they respawn at the insertion if another player hasn't already broken it, however, GreenPls wanted the ability to place multiple insertions as breakable warp points for his factions server. This is where warp mode comes in. Warp mode allows players to place multiple insertions, name them, and then wrap to them at any time until they are broken by another player.

How to use Tactical Insertions

To get an insertion, run the command /gettac, for get tactical. Place the insertion far enough from others (the plugin will warn the player if it isn't). Warp mode will then have the player name the insertion while respawn mode will be done.

Warp Mode

Warp mode is GreenPls's original idea for the plugin. Once the player has named the insertion after placing it, it will become a location he or she can warp to. To find all of the warps available to him or her, run the command /tacwarps. This will list all of his or her available warps. To warp to them, run /tacwarp , replacing name with the name of the warp.

Respawn Mode

When the player dies, he or she will respawn at their tactical insertion. It will break once the player respawns.

Server Owner's Instructions

All Commands

  • tacwarp : Warps the player to his or her insertion with the same name. This is for warp mode only.
  • tacwarps: Lists all of a player's warps. This is for warp mode only.
  • ti: Contains moderator commands (is not a command itself).
  • -ti version: Prints the plugin version
  • -ti changemode: Changes the plugin from one mode to the other. The server must be restarted or reloaded after this command is run, as Tactical Insertions will shut down.
  • -ti clearinsertions: Deletes all insertions from the server. This must be done before changing modes.

Permissions

  • tacticalinsertions.getblock: Lets the player use /gettac
  • tacticalinsertions.placeblock: Lets the player place tactical insertions
  • tacticalinsertions.moderator: Gives access to the moderator commands.

Changing the plugin mode

To change the plugin mode, you must first clear all insertions from the server using /ti clearinsertions. You will then be able to run /ti changemode. The plugin will shut down after this. Restart or reload the server to reenable the plugin. This is necessary to prevent the numerous problems that could come from such a dramatic change the plugin would have to go through.

Language Files

Want to translate TacticalInsertions to another language or just want to change some of the messages? By changing the language files inside the Language folder the plugin creates, you can change the messages from the plugin. If you translate this to another language, please send me the file with the translations. I will put the translation into the plugin officially for others to use!

The Configuration File

Here are the fields of the configuration file and their explanations.

  • plugin_prefix: the name the plugin uses when talking to players
  • distance_from_other_tac_minimum: the number of blocks away an insertion has to be from others
  • allow_players_to_drop_insertions: whether or not dropped insertion blocks can exist in the world
  • tac_block_material: the material of the insertion block. If you change this and you can no longer get an insertion, it is invalid. Typically, solid blocks are your best bet. Use the material list https://hub.spigotmc.org/javadocs/spigot/org/bukkit/Material.html to select a material. It is case sensitive.
